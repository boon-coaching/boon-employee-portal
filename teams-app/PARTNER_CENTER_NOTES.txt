RESUBMISSION SUMMARY — HOW EACH ISSUE WAS ADDRESSED (v1.0.7)
================================================================

v1.0.7 delta vs v1.0.6:
- Fixes Test Blocker #6 (continuous loading screen on Connect
  Microsoft Teams). Root cause: the OAuth callback was serializing
  the employee_teams_connections save behind a best-effort Bot
  Framework "create proactive 1:1 conversation" call. On tenants
  where the bot app is not yet installed on the user's personal
  scope, that Bot Framework call could stall long enough for the
  function to redirect before the connection row was persisted,
  leaving the portal in a not-connected state.
- Fix: the employee_teams_connections row is now upserted
  immediately after the Microsoft Graph profile lookup, with an
  empty conversation_id. The proactive conversation creation is
  then attempted with a 7-second timeout; on success the row is
  updated with the conversation_id. On timeout or failure, the
  conversationUpdate handler backfills conversation_id + service
  URL as soon as the bot is added to the user's personal chat
  (using activity.channelData.tenant.id + activity.from.aadObjectId
  / membersAdded[].aadObjectId as the match keys).
- UX: the portal Settings page now shows a visible success or
  error toast after the OAuth redirect and retries the connection
  status up to 3 times on a brief read-replica lag, so the
  "Connected via Microsoft Teams" badge appears deterministically.

1. End-to-end test path missing: this notes block now provides a
   numbered Path A and Path B walkthrough. Path A uses the
   M365x34009223 test tenant you provided, with the app already
   side-loaded by us so the bot is available to adelev@... before
   testing begins.

2. "Only Slack option in portal, no Teams option": fixed. Settings
   page at https://my.boon-health.com/settings now shows both
   "Connect Slack" and "Connect Microsoft Teams" buttons.
   One-channel-at-a-time enforced server-side.

3. Screenshot captions: captions are provided via the alt text on
   each screenshot in the Partner Center listing.

4. New-user contact / help / sign-up access: provided in three
   places — (a) Welcome Adaptive Card on bot install includes
   Open Boon Portal, Get Help, and Contact Us buttons; (b) long
   description includes Sign Up, Get Started, Help, Contact Us
   links; (c) portal Settings page links to Help and Contact.

5. Long description rewritten to include features, target audience,
   account dependency, sign-up/help/contact links, and limitations.

6. Welcome bot message: bot now sends a proactive Adaptive Card
   on conversationUpdate when added to a personal chat. Card
   includes feature overview and three action buttons.

7. Hi / Hello / Help / fallback responses: bot responds to hi,
   hello, hey, help, menu, start, get started with the Welcome
   Adaptive Card. Any other text returns a fallback card with
   forward actions. No dead ends.

Items noted as recommendations in the prior validation (mobile
screenshots, promotional video, Publisher Attestation) are
acknowledged; they were flagged as enhancements rather than
blockers and are being tracked on our roadmap.
